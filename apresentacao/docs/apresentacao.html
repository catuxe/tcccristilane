DESENVOLVIMENTO DE UM PROTÓTIPO DE SISTEMA PARA APOIO ÀS ATIVIDADES PSICOPEDAGÓGICAS NO INSTITUTO FEDERAL DE SERGIPE

Discente: Cristilane Gois de Souza
Orientador: Telmo Oliveira de Jesus
Coorientadora: Catuxe Varjão de Santana Oliveira

29 de janeiro de 2026

Contextualização do Estudo

  • Psicopedagogia nas instituições escolares
  • Educação inclusiva
  • Atendimento psicopedagógico no Instituto Federal de Sergipe-IFS, Campus Lagarto
  • Atuação do Núcleo de Atendimento às Pessoas com Necessidades Específicas-NAPNE
  • Necessidade de organização das informações

Problema Investigado

  • Registros psicopedagógicos realizados de forma manual
  • Informações dispersas
  • Dificuldade no acompanhamento da evolução dos alunos
  • Fragilidade na comunicação entre:
    • Instituição
    • Equipe multiprofissional
    • Família

Objetivo Geral

  • Desenvolver um protótipo de sistema para apoiar as atividades psicopedagógicas no IFS, contribuindo para a organização dos atendimentos, o acompanhamento dos estudantes e o fortalecimento da comunicação entre instituição, psicopedagogo e família.

Objetivos Específicos

  • Levantar as necessidades do atendimento psicopedagógico institucional
  • Definir as funcionalidades do sistema de acordo com essas necessidades
  • Modelar o sistema por meio de diagramas de caso de uso, sequência e atividades.
  • Desenvolver um protótipo que auxilie no registro e acompanhamento dos atendimentos psicopedagógicos.
  • Contribuir para a organização das informações e a comunicação com os pais ou responsáveis.

Requisitos Funcionais

Código Requisitos Funcionais Descrição
RF01 Cadastrar aluno O sistema deve permitir o cadastro de alunos, registrando suas informações pessoais e educacionais.
RF02 Visualizar dados do aluno O sistema deve permitir a visualização dos dados completos dos alunos cadastrados.
RF05 Listar e filtrar alunos O sistema deve permitir a listagem de alunos cadastrados, com aplicação de filtros para facilitar a busca.
RF12 Gerar documentos e encaminhamentos O sistema deve permitir a geração de relatórios, encaminhamentos e termos relacionados ao atendimento do aluno.
RF14 Registrar anamnese O sistema deve permitir o registro da anamnese do aluno e do responsável.
RF16 Agendar atendimento O sistema deve permitir o agendamento de atendimentos.
RF22 Notificar equipe multiprofissional O sistema deve permitir o envio de notificações à equipe multiprofissional sobre o andamento e a conduta do aluno.
RF25 Registrar devolutivas clínicas O sistema deve permitir o registro de devolutivas clínicas destinadas às famílias e à equipe multiprofissional.
RF28 Agendar atendimento O sistema deve permitir construir PEI em parceria com a equipe Multiprofissional.

Requisitos Não Funcionais

Código Requisitos Não Funcionais Descrição
RNF01 Segurança O sistema deve garantir autenticação de usuários por meio de login e senha.
RNF04 Usabilidade O sistema deve possuir interface intuitiva e de fácil navegação.
RNF06 Desempenho O sistema deve responder às requisições dos usuários em tempo adequado.
RNF08 Confiabilidade O sistema deve evitar a perda de dados em caso de falhas.
RNF11 Manutenibilidade O sistema deve permitir manutenção e atualização sem comprometer seu funcionamento.
RNF12 Legalidade O sistema deve estar em conformidade com a legislação vigente sobre proteção de dados pessoais.
